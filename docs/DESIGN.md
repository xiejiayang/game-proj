# 都江堰水利工程主题解谜游戏 — UI/UX 设计规范

> 版本：V0.1  
> 日期：2026-06-25  
> 适用范围：Vertical Slice（L1）及后续 v1.0  
> 目标：为 Unity uGUI/UI Toolkit 实现提供统一的设计约束

---

## 1. 设计原则

### 1.1 整体方向
- **视觉先行**：解谜信息优先通过视觉、动画、空间关系传递，文字仅做"命名"。
- **水墨淡彩**：宣纸底、墨线、青绿、赭红构成主视觉语言。
- **文化不说教**：界面本身即文化表达，不堆砌文字说明。
- **零惩罚友好**：失败反馈诚实但不挫败，重试成本极低。

### 1.2 设计目标
- 玩家第一眼能识别出"水墨风格"。
- 玩家在 5 秒内理解当前所处阶段（编辑/模拟/结算）。
- 所有交互反馈在 200ms 内发生。

## 2. 色彩系统

### 2.1 主色板

| 名称 | 色值 | 用途 |
|------|------|------|
| `--ink-dark` | `#1a1a1a` | 主文字、主边框、按钮边框、标题 |
| `--ink-mid` | `#4a4a4a` | 次级文字、说明文案 |
| `--ink-light` | `#8a8a8a` | 禁用态、占位文字、弱化信息 |
| `--paper` | `#f7f3e9` | 全局背景、卡片底色 |
| `--paper-dark` | `#e8e0d0` | 卡片悬浮态、分隔、暗部背景 |
| `--accent` | `#2d5a4a` | 主强调色、成功、放水按钮、选中态 |
| `--accent-light` | `#4a8a72` | 悬停态、高亮、进度条填充 |
| `--danger` | `#8b3a3a` | 失败、危险、重置按钮 |
| `--water` | `#3a5a6a` | 水流、时间资源、水源标识 |
| `--empty` | `#b8b8b8` | 空数据、占位符、禁用图标 |
| `--skeleton` | `#e0d8c8` | 加载骨架、未解锁内容遮罩 |

### 2.2 使用规则
- 背景只用 `--paper` 或 `--paper-dark`。
- 文字层级：标题用 `--ink-dark`，正文用 `--ink-mid`，提示/禁用用 `--ink-light`。
- 成功态以 `--accent` 为主，失败态以 `--danger` 为主，不允许互换。
- 水流相关元素统一使用 `--water` 色系。

### 2.3 透明度规范
| 层级 | 推荐透明度 |
|------|-----------|
| 背景纹理 | 6% |
| 卡片背景 | 95% |
| 按钮 hover 背景填充 | 100% |
| 遮罩层 | 40%–95% |
| 构件半透明预览 | 50% |

## 3. 字体系统

### 3.1 字体栈
```css
font-family: "Noto Serif SC", "Source Han Serif SC", "SimSun", serif;
```

### 3.2 字体规则
- **使用免费开源字体**：优先使用 Noto Serif SC / Source Han Serif SC（OFL 协议），避免商业书法字体版权风险。
- **单字体策略**：Vertical Slice 只使用一种字体家族，通过字号、字重、字间距区分层级。
- **不回退到版权不明字体**：如果目标平台没有 Noto Serif SC，回退到系统默认衬线字体，不使用需要额外授权的商业字体。

### 3.3 字号层级

| 层级 | 字号 | 字间距 | 用途 |
|------|------|--------|------|
| H1 | 32px | 6px | 游戏标题 |
| H2 | 24px | 4px | 关卡标题、页面标题 |
| H3 | 20px | 2px | 卡片标题、结算大字 |
| Body | 16px | 0 | 正文、按钮文字 |
| Caption | 14px | 0 | 说明、资源数字 |
| Label | 12px | 0 | HUD 标签、小计数 |
| Seal | 32px | 0 | 印章大字 |

### 3.4 字体渲染
- Unity 中字体材质使用 SDF（Signed Distance Field）以保证放大后边缘清晰。
- 字号小于 14px 时开启字体Hinting，避免发虚。

## 4. 布局与间距

### 4.1 设计画布
- 移动端优先设计尺寸：**375 × 812 px**（iPhone 标准逻辑分辨率）。
- 大屏适配：居中显示，最大宽度 430px，两侧留白。
- 安全区：顶部预留 44px，底部预留 34px（适配刘海/圆角）。

### 4.2 间距系统
以 **8px** 为基准网格，间距取值为：
```
4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 px
```

### 4.3 边距规则
- 屏幕内边距：16px（左右），12px–16px（上下）。
- 卡片内边距：24px。
- 按钮内边距：14px 垂直，48px 水平（主按钮）；8px 垂直，24px 水平（小按钮）。
- 组件间距：12px（紧凑），16px（标准），24px（宽松）。

## 5. 组件规范

### 5.1 BtnPrimary
- 背景：`--ink-dark`
- 文字：`--paper`，18px，字间距 4px
- 圆角：0（方正水墨感）
- 内边距：14px 48px
- Hover：背景变为 `--ink-mid`
- Press：缩放 0.97，持续 100ms
- 使用场景：开始治水、放水、下一关

### 5.2 BtnSecondary
- 背景：透明
- 边框：2px solid `--ink-dark`
- 文字：`--ink-dark`，18px
- Hover：背景填充 `--ink-dark`，文字变白
- 使用场景：设置、返回标题、查看碑廊

### 5.3 BtnGhost
- 背景：透明
- 边框：1px solid `--ink-mid` 或无框
- 文字：`--ink-mid`，13px–14px
- Hover：文字变为 `--ink-dark`
- 使用场景：撤销、重置、跳过

### 5.4 Card
- 背景：`--paper`
- 边框：2px–3px solid `--ink-dark`
- 圆角：16px（水墨卡片柔和感）
- 内边距：24px
- 阴影：不使用硬阴影，仅用边框和轻微宣纸纹理
- 使用场景：提示卡、结算卡、设置项容器

### 5.5 ToolbarItem
- 尺寸：60 × 60px
- 背景：`rgba(255,255,255,0.4)`
- 边框：2px solid `--ink-dark`
- 圆角：12px
- 选中态：背景 `--accent` 15% 透明度，边框 `--accent`
- 禁用态：透明度 40%
- 子元素：图标 22px + 计数 11px

### 5.6 ResourceDot
- 尺寸：8 × 8px
- 圆角：50%
- 颜色：料 `--accent`、工 `--danger`、时 `--water`
- 与文字间距：4px

### 5.7 Seal（印章）
- 尺寸：90 × 90px（结果弹窗）/ 可缩放
- 边框：4px solid（成功 `--accent`，失败 `--danger`）
- 圆角：8px
- 文字：32px，粗体，旋转 -5°
- 成功文字：「安」
- 失败文字：「倒」
- 动画：旋转落下 + 轻微弹起，500ms

### 5.8 DialogOverlay
- 背景：`rgba(0,0,0,0.5)`（普通遮罩）或 `rgba(247,243,233,0.95)`（暂停遮罩）
- 对齐：居中
- 内边距：24px
- 不允许点击外部关闭（除 HintPill 外）

### 5.9 HintPill
- 背景：`rgba(247, 243, 233, 0.95)`
- 边框：1px solid `--ink-dark`
- 圆角：20px（胶囊）
- 内边距：10px 16px
- 字体：13px
- 行为：自动消失，2.5s

## 6. 动效与交互

### 6.1 动效原则
- **叙事场景慢**：引子、结算、碑廊使用 400–600ms 水墨优雅过渡。
- **操作反馈快**：拖拽、点击、放置使用 100–200ms 即时反馈。
- **不阻塞操作**：动画播放期间，关键交互（重试、返回）保持可用。

### 6.2 具体动效规范

| 场景 | 时长 | 曲线 | 效果 |
|------|------|------|------|
| 页面切换 | 400ms | ease-in-out | 淡入 + 向上位移 12px |
| 按钮点击 | 100ms | ease-out | 缩放 0.97 或背景色填充 |
| 拖拽 ghost 出现 | 0ms | - | 立即跟随手指 |
| 构件吸附网格 | 150ms | ease-out | 从释放位置插值到网格点 |
| 放水按钮变化 | 200ms | ease-in | 文案变为「模拟中…」，按钮 disabled |
| 水粒子生成 | - | - | 水源持续发射，速度越快墨色越浓 |
| 构件被冲毁 | 300ms | ease-in | 透明度降为 0，可选碎石飞散 |
| 印章落下 | 500ms | cubic-bezier(0.34, 1.56, 0.64, 1) | 旋转 -5° 落下 + 弹起 |
| 老河工提示 | 400ms | ease-out | 从底部滑入 + 淡入 |
| 结算页切换 | 300ms | ease-in-out | 屏1 淡出，屏2 淡入 |
| 暂停菜单出现 | 200ms | ease-out | 遮罩淡入 + 按钮依次上移 50ms stagger |

### 6.3 拖拽交互细节
- 手指按住 ToolbarItem 时立即显示 ghost。
- ghost 中心跟随手指。
- 释放位置在 canvas 内：构件吸附到最近网格点，播放 150ms 放置动画。
- 释放位置在 canvas 外：ghost 淡出，不消耗库存。
- 放置非法位置（河道外/重叠）：构件变红 200ms 后回到 toolbar，不消耗库存。

### 6.4 状态切换视觉
- 编辑阶段：河道网格可见，Toolbar 可用，「放水」按钮高亮。
- 模拟阶段：网格隐藏，Toolbar 禁用，「放水」按钮变为「模拟中…」并灰显。
- 结算阶段：弹出结果卡片，背景变暗。

## 7. 状态视觉规范

### 7.1 加载中
- 背景：`--paper`
- 元素：水墨进度条 + 文案「研墨中…」
- 进度条：高度 4px，填充 `--accent-light`，背景 `--skeleton`
- 可选文案轮播：「研墨中…」「引水入田…」「加固堤岸…」
- 不显示具体百分比（避免焦虑），但进度条必须真实反映加载进度。

### 7.2 空操作
- 触发：未放构件点击放水
- 表现：顶部 HintPill「先在河里摆上构件」
- 持续：2.5s 后自动淡出
- 不阻断玩家继续操作

### 7.3 提示中
- 触发：老河工给出指引
- 表现：底部居中对话框，老河工头像 + 文字
- 文字出现方式：打字机效果，每字 40ms
- 关闭：点击「知道了」或 5s 后自动关闭

### 7.4 暂停
- 触发：点击暂停按钮
- 表现：全屏 `--paper` 95% 不透明遮罩
- 标题：「已暂停」，28px
- 按钮：继续、重试、设置、返回标题，竖排 16px 间距

### 7.5 成功
- 触发：村庄未被淹没
- 第一屏：
  - 印章「安」，绿色
  - 标题「暂时安全」
  - 描述：根据是否节俭显示不同文案
  - 按钮：「查看碑廊」
- 第二屏：
  - 碑廊标题「堵不如疏」
  - 图文内容
  - 按钮：「再试一次」（隐藏）+「下一关」

### 7.6 失败
- 触发：村庄被淹没
- 第一屏：
  - 印章「倒」，红色
  - 标题「村子仍被淹」
  - 描述：提示调整思路
  - 背景：村庄变暗或水面上升
  - 按钮：「再试一次」
- 不显示「下一关」按钮

### 7.7 错误
- 触发：WebGL 不支持、存档失败、资源加载失败
- 表现：简单弹窗
- 标题：「无法运行」/「出错了」
- 说明：简短解释原因
- 按钮：「重试」/「返回」

### 7.8 空数据/未解锁
- 触发：碑廊中未解锁条目
- 表现：卡片占位，显示锁图标 +「通关后解锁」
- 颜色：`--empty`

## 8. 响应式与安全区域

### 8.1 响应式策略
- 以 375px 宽度为基准设计。
- 宽度 > 430px 时，游戏画布居中，两侧显示宣纸纹理背景。
- 高度 < 700px 时，Toolbar 高度压缩至 120px，HUD 高度压缩至 100px。

### 8.2 安全区域
- 顶部：44px（状态栏/刘海）
- 底部：34px（圆角/手势条）
- HUD 顶部元素避开安全区顶部。
- Toolbar 底部元素避开安全区底部。

## 9. 视觉资产规范

### 9.1 3D 场景
- 低多边形风格，面数控制在 500 以内/模型。
- 色调：淡墨 + 青绿，饱和度不超过 40%。
- 避免高反光、写实材质。

### 9.2 水流/粒子
- 粒子使用 Trail Renderer + 水墨 Shader。
- 速度越快，墨色越深、拖尾越长。
- 粒子数量在 WebGL 上控制在 200 以内。

### 9.3 2D 人物/叙事
- 使用 AI 生成水墨风格剪影/立绘。
- 统一关键词：水墨、淡彩、宣纸、宋代山水、留白。
- 存放于 `Assets/_Project/Art/AI_Generated/`，附带元数据文件。

### 9.4 UI 纹理
- 优先使用程序生成的宣纸噪点，减少贴图数量。
- 如需贴图，尺寸为 2 的幂次方，最大 1024px。
